Projects

Retro Multi-player Web-based Dungeon Crawler Game

This is a class assignment for my Software Systems class (winter 2017)

The player starts out in a room with n gold and n mana. Players type into the text box for events to happen. The C program I wrote parses the query string that is passed via the POST and GET methods from a form, commands that are recognized are then executed. A program called transporter.py takes the player to a different room, bringing with them their gold and mana. One trip on the transporter costs 1 mana. The transporter program is responsible for calling the other rooms "refresh" command.

Every room has a challenge and when players win the challenge they get to take gold and mana from the rooms hidden resources (a resources.csv file). The sum of gold and mana a player can pick up after winning a challenge is 5. A player dies when his/her mana drops to 0, and wins when they obtain 100 gold.

Commands supported for this game are:

- Drop n: to drop gold. For every 2 gold that is dropped the user gains 1 mana

- Refresh: redraws the page with the players inventory

- Exit: player exits the room, leaving behind their inventory that gets added to the rooms hidden resources.

- Play: The player is brought to a page which prompts him/her the option to play the game. The player has a choice between playing the game and not playing the game (which would bring them back to the default page)

- Bother: bothers one of the NPCs. Doing so would result in the player losing 2 mana

- Suicide: a command that takes all the players mana and shows the "game over" screen

Created with: CGI Programming in C & Python HTML CSS Javascript
